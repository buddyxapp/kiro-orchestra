Features

📋
Task View
Multi-step tasks with stages, progress tracking, approve/reopen/archive. Scope guard prevents cross-task interference.
🧠
Event-Driven Master
Stays responsive while workers are busy. Accepts new input anytime. Manages multiple concurrent tasks.
📚
LLM Wiki (Memory)
Per-task report folders. Long results saved to .md files. Master sees summaries, reads full files when needed. Survives restart.
📄
HTML Report Execute
Reports with action buttons. User checks items → clicks Execute → Orchestra processes. Works from local file:// pages.
⚡
Parallel Execution
Multiple workers run tasks simultaneously. Each can use a different AI model.
🎯
@mention Routing
Direct messages to specific agents, broadcast to all, or channel-aware auto-routing.
⏸️
Cancel & Control
Stop button, per-agent cancel, TASK_WAIT for user decisions. Master can't close tasks without user approval.
📷
Image Paste
Ctrl+V screenshots into chat. Saved to file, agents analyze via read tool.
📊
Activity Panel
Right-side panel shows real-time tool calls per agent. Chat stays clean.
🏠
100% Local
Runs on localhost. No external services. No API keys beyond kiro-cli. All data in wikis/ (gitignored).
